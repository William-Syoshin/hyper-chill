# コントリビューションガイド

HyperSystemへの貢献を歓迎します！

## 開発環境のセットアップ

1. リポジトリをフォーク
2. ローカルにクローン
3. 依存パッケージをインストール: `npm install`
4. `.env.local` を設定
5. 開発サーバーを起動: `npm run dev`

## ブランチ戦略

- `main`: 本番環境用の安定版
- `develop`: 開発中の機能
- `feature/*`: 新機能開発用
- `bugfix/*`: バグ修正用

## コミットメッセージ

以下の形式に従ってください：

```
<type>: <subject>

<body>
```

**Types:**
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメント更新
- `style`: コードスタイルの変更
- `refactor`: リファクタリング
- `test`: テスト追加/修正
- `chore`: ビルドプロセスやツールの変更

**例:**
```
feat: 写真一括ダウンロード機能を追加

管理画面から承認済み写真を一括でダウンロードできる
機能を追加しました。
```

## コーディング規約

### TypeScript

- 型を明示的に定義
- `any` の使用は避ける
- ESLintのルールに従う

### React

- 関数コンポーネントを使用
- Hooksを適切に使用
- propsの型を定義

### ファイル構成

- コンポーネントは `components/` 配下
- Server Actionsは `actions/` 配下
- ユーティリティは `lib/` 配下

## プルリクエスト

1. 機能ブランチを作成
2. 変更をコミット
3. テストを実行
4. プルリクエストを作成
5. レビューを待つ
